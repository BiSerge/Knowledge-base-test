Обновление платформы. Совместное использование 8.1 и 8.2 на одном сервере

Внимание! Журнал регистрации после обновления платформы будет утерян!! Для сохранения нужно его предварительно сохранить.

Установка сервера 1С 8.2 рядом с сервером 1С 8.1

Программа установки 1С:Предприятия настраивает сервер 1С:Предприятия на использование стандартных портов 1540, 1541, 1560-1591. Если после установки сервер 1С:Предприятия 8.2 должен работать одновременно с сервером 1С:Предприятия 8.1, то он должен использовать порты, отличающиеся от портов уже работающих экземпляров сервера. Поэтому используемые экземпляром сервера порты необходимо изменить.Если у Вас уже установлен сервер 8.1, то нужно установить сервер 8.2 по стандартной методике, но в итоге сервер 8.2 не будет работать.
Открываем редактор реестра — команда REGEDIT, находим ветку реестра [HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\1C:Enterprise 8.2 Server Agent]. Параметр ImagePath будет выглядеть так :

«C:\Program Files(x86)\1cv82\8.2.15.301\bin\ragent.exe» -srvc -agent -regport 1541 -port 1540 -range 1560:1591 -d «D:\Program Files (x86)\1cv82\srvinfo»,

его нужно изменить на:

«C:\Program Files (x86)\1cv82\8.2.15.301\bin\ragent.exe» -srvc -agent -regport 1641 -port 1640 -range 1660:1691 -d «D:\Program Files (x86)\1cv82\srvinfo»

меняем номера портов с 15*.. на 16*…

Внимание! В строке -d «D:\Program Files (x86)\1cv82\srvinfo» изменить диск C на диск D - это изменить место хранения логов и индексов. Проверить эту сроку на базе 8.1

Нужно удалить содержимое каталога «C:\Program Files\1cv82\srvinfo». После этого желательно перезагрузить машину.

После перезагрузки можно проверить в консоли «Управление компьютером — Службы» что запущено два «Агента сервера 1С:Предприятия» 8.1 и 8.2. Открываем консоль «Администрирование серверов 1С Предприятия», будьте внимательны теперь у Вас две консоли 8.1 и 8.2.

  • Создаем сервер с именем компьютера и портом 1640
  • Создаем кластер с портом 1641 (если его нет)
  • Создаем Рабочий сервер с портом 1640 и диапазоном 1660:1691 (если его нет)
  • Создаем Рабочий процесс

Внимание
При добавлении базы в окне запуска 1С при записи Кластера серверов нужно указывать порт Пример: Имя_Компьютера:1641
